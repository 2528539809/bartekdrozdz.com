Scrolling and clicking (or tapping) are the two most popular ways pepople interact with the web. Clicking has been traditionally used for navigation and moving between pages, while scrolling was used to move within the content of a single page. Nowadays this distinction is often blurred. Scrolling is used to move between sections, while clicking on buttons and links doesn't reload the page but makes it scroll instead.

History of scrolling

For may years scrolling was on the web was straightforward. If a page was longer than the browser height, user could move it using his mouse wheel, touch pad or by clicking and dragging the scroll bar located on the right side of the window. When Javascript started to be used more often, especially jQuery, things started to get a bit more fancy, with things like fixed navigation, sticky buttons or animated panels triggered when the user would scroll beyond a certain threshold.

Then came paralax scrolling. It has been, for better or for worse, one of the dominating interactive desing trends in the last years. It started with an effect where things would move at different pace when the user scrolls, creating a nice impression of depth and perspective. This is where the name comes from - "paralax" is the term used to describe the effect where things further away from the viewer move slower than things closer to him as he moves by.

During the paralax euforia, website creators went on and on, creating more elaborate scrolling schemes. Some websites had elements that on scroll moved literally in every possible direction. The effect was usually terribly chaotic.

Later on "scroll hijacking" was introduced. It is a technique that consists of replacing the entire content of the page when the user scroll up or down. It is a very tricky thecnique, because it breaks the fundamental way scrolling works with its analog and continous nature. That is why the majority of these websites offer a poor user experience.

While the web was dealing with these unfortunate trends, more interesing uses of scrolling came from the mobile OS world. Apple intorduced the elastic overscroll effect first as a toy (addictive toy, I might add) and then introduced the "pull to refresh" function which is one of the most innovative uses of the scroll mechanism.
